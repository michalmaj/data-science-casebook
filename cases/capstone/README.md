# Capstone — Choose Your Own Engagement

Every previous case told you who the client was, what they wanted, and roughly what to build. This time you pick.

## The menu

Three prospective clients, each with a light brief. Pick one — you'll work with it for all six lessons of the capstone.

### Riverside Community Clinic (`clinic_wait_times.csv`)

"Patients keep telling us the wait feels endless, but we don't actually know what drives it or how to set expectations."

### LendWell Financial (`lendwell_loan_default.csv`)

"We approve loans without a systematic way to gauge risk, and it's starting to cost us."

### Northfield Retail Group (`retail_store_segments.csv`)

"Our stores aren't all the same, but we treat them the same in regional planning. We suspect there are natural groups here — we just don't know what they are."

None of these briefs tell you the target variable, the success metric, or the exact question to answer. That's your job, starting in Lesson 1.

## What's different about the capstone

- **You pick the dataset.** Every lesson's code works generically across all three, so whichever one you choose, the tools you're given will work.
- **You define the problem.** Lesson 1 is where you turn a vague client complaint into a specific, measurable analytical question.
- **The self-checks are looser.** `check.py` in these lessons checks structure and reasonableness (does your function return the right shape, does your model expose `.predict()`, is your metric in a sane range) — not exact numbers, because there's no single correct answer here.

## Data format

Plain CSV, one file per client, generated by `data/generate.py`.

## Grading

Same rubric as every case in this repo (see [`ASSESSMENT_RUBRIC.md`](../../ASSESSMENT_RUBRIC.md)): problem definition 15%, data recognition/quality 15%, exploration/justification 20%, modeling/evaluation correctness 20%, interpretation/limitations 15%, communication 10%, readability/reproducibility 5% — applied to your final decision note (Lesson 6). See [`lessons/06_synthesis_decision_note/exemplar_decision_note.md`](lessons/06_synthesis_decision_note/exemplar_decision_note.md) for a model answer.

Lessons live under `lessons/`, numbered in the order you should work through them.
